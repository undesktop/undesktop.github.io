    Mission

    The Undesktop Project exists to decouple desktop applications from the desktop environment. We work towards this by developing high-quality, modern desktop applications which work on their own.

    Technology

    We understand that making a truly great application requires the best technology. For this reason, we have compiled a shortlist of preferred technologies. Other technology may be used when required.

    • Vala -- as deep as possible
    • C -- never C++, ever
    • CMake -- to compile everything
    • GTK3 -- for anything with a GUI
    • Clutter -- minimized to avoid problems in software rendered environments
    • GObject -- vala makes this easy

    Code Standards

    • Follow the division of model/view/controller. The view should emit signals which the controller then listens to and calls methods on the model.
    • Use GResources to compile in things like your .ui files, which don't need to be changed by the end user or by packagers.
    • Indent with tabs, align with spaces. For explanation of my reasoning, see Lea Verou's post on the topic. She was 100% spot-on.
    • Don't write a manpage in groff, seriously. Nobody fucking knows groff. Use pandoc markdown instead: we have a CMake task which compiles pandoc into groff for manpages.
    • Don't play favorites with backends. If there's like 4 backends (like for network or audio), don't just pick one and use it. Provide an interface which the backend classes can implement.
    • Handle your errors as locally as possible, don't let them bubble far.
    • Mark your nullable types in Vala, we use --enable-experimental-non-null. Follow this even when you have to disable that flag to use code which doesn't mark nullable types.
    • Test your code. It doesn't have to be perfect, but at least try to get good coverage.
    • We use TravisCI

    The Future
